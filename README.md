# 周祥 / euiwpoi

我在搭一套**本地优先、能长期跑起来的 AI 工作系统**。

重点不是聊天效果，而是让 AI 真正参与：

- 写作
- 整理
- 执行
- 复盘

## 现在主要在做

- 长篇小说 / 短篇小说工作流
- 浏览器自动化与个人代理流程
- 本地资料整理、知识库和归档
- 重复任务脚本化

## 计划公开的仓库

- `fiction-writing-ops`
  - 小说 / 剧本工作流、模板、检查表
- `browser-ops`
  - 浏览器自动化、专用配置、执行脚本
- `agent-ops`
  - 个人 AI 代理流程、任务拆分和执行规范
- `local-knowledge-workbench`
  - 本地资料整理、索引、归档和检索工具

## 做事方式

- 先解决问题，再谈形式
- 能本地就尽量不依赖云端
- 能复用就不重复劳动
- 工具必须真的省时间

## 这里后面会放什么

- 可公开的工作流文档
- 能直接拿来用的脚本
- 已验证过的实用方案
- 个人项目里适合公开的部分

## 一句话说明

如果一个工具只是看起来聪明，但不能稳定把事情做完，我一般不会长期用它。
